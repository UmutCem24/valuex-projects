      Methodology · how the ranking works

      The script summary below is the entire evaluation logic in pseudocode form — what runs every time you press Run. The sections below it expand each step. This replaces the manual Sintra → Excel process: same 7 dimensions, same 17 criteria, same 200 threshold, now computed consistently and traceably.

      Evaluation Script · özet prompt v1.0 · profile-aware · 0–360 scale 
      // INPUT — every project record
read  typeform_answers[17]          // founder self-assessment Q&A
read  pitch_deck.pdf                // page-parsed, source-tagged deck_pN
read  website_url                   // fetched
read  documentation_links[ ]
read  uploaded_files[ ]             // financial model, audit, tokenomics, ...
read  external_signals              // linkedin, github, discord, on-chain
read  internal_notes
set   profile ∈ { TECHX, WEB3X }   // chosen at intake

// SCORE — 17 criteria, 1..6 each
for c in 1..17:
  if profile == TECHX and c in {9, 10}:  skip          // token criteria
  score[c]      ← VALUEX.score(c)                // strict, evidence-based 1..6
  confidence[c] ← VALUEX.confidence(c)           // Low | Medium | High
  evidence[c]   ← VALUEX.tier(c)                 // Verified | Supported | Inferred | Claim
  citations[c]  ← VALUEX.cite(c)                 // [tf_q05, deck_p12, audit.pdf, ...]

// HARD SCORING RULES
rule  claim_without_artifact   →  cap score at 4
rule  hype_words               →  do not lift score
rule  contradiction_present    →  push score down
rule  stage_aware              →  pre-revenue ≠ scaling

// COMPUTE TOTAL — 7 dimensions roll up
for d in 7_dimensions:
  contribution[d] ← (Σ score[c ∈ d]) × weight[d, profile] × 100
total ← Σ contribution[d]                        // range 0..360 (200 = listing threshold)

// PLAUSIBILITY PASSES — what reviewer used to do by hand
contradictions ← detect_conflicts(all_inputs)    // deck ↔ form ↔ external
missing        ← detect_missing(criteria, stage, sector)
red_flags      ← detect_red_flags(all_inputs)

// ROUTE DECISION
if critical_contradiction or critical_missing:
  recommendation ← "CLARIFICATION_NEEDED"           // override regardless of score
elif total ≥ 241:
  recommendation ← "LIST"                          // top / elite
elif total ∈ [200, 240]:
  recommendation ← "HUMAN_REVIEW"                  // borderline — reviewer signs off
elif total ∈ [121, 199]:
  recommendation ← "REVISIT_LATER"                 // 6-month timer + workshop offer
else:
  recommendation ← "NOT_A_FIT"

// OUTPUT — renders into workstation tabs
return {
  scores[17], confidence[17], evidence[17], citations[17],
  contradictions[ ], missing[ ], red_flags[ ],
  dimension_contributions[7], total, band, recommendation,
  top_5_strengths, top_5_weaknesses,
  founder_facing_summary
}                                                // → Overview · Scoring · Evidence · Analytics · Decision

      1 What goes in

      VALUEX evaluates the full submission package — not just the deck. Each input is source-tagged so every score can be traced back to where the evidence came from.

      Inputs   Typeform answers (the 17 self-assessment questions) · pitch deck (parsed page by page) · website (fetched) · documentation & data-room links · uploaded files (financial model, audit, tokenomics) · external public signals where reachable (LinkedIn, GitHub, Discord, block explorers) · internal reviewer notes. Each gets a tag like tf_q05, deck_p12, audit.pdf so citations link back to the source.

      3 How each 1–6 score is decided

      Evidence-based, investor-minded scoring — fair rather than punitive. The score is "how well-proven is this, against a bar appropriate to the project's stage," with the benefit of the doubt going to a coherent, good-faith submission. The same anchors apply to every project so rankings stay comparable.

      1 – 2
      Absent, contradicted, or a genuine red flag — not merely the incompleteness typical of an early-stage project.
      Weak
      3 – 4
      Plausible and reasonably evidenced for the project's stage. This is the default landing zone for a competent, good-faith submission.
      Adequate / Good
      5 – 6
      Strong and well-evidenced — externally verifiable or backed by a working artifact.
      Strong
      Hard rules   A coherent claim with no supporting artifact is capped at 4, not lower. Contradictions push the score down. Pre-revenue stage is not penalised for zero revenue (stage-aware), but a "Scaling" stage with zero revenue is. Hype words never lift a score on their own — but a specific, internally-consistent narrative counts in the project's favor.

      4 Evidence tier — claim vs proof

      This is the biggest upgrade over the old process. Every score also carries an evidence tier, so you can instantly see whether a high score is proven or merely asserted.

      Verified
      Externally checkable & third-party confirmed — audit report, on-chain data, SEC filing, named investor with public round.
      Supported
      Backed by an uploaded artifact or working link — financial model, demo URL, whitepaper. Not yet externally verified.
      Inferred
      Plausible from adjacent signals — team strength inferred from LinkedIn, market timing from sector activity.
      Founder Claim
      Asserted in the form or deck only, with no supporting artifact. Treated with most caution.

      6 The formula

      Each dimension takes the average of its criterion scores (not the sum), multiplies it by the dimension weight, and by 60. The seven dimension contributions sum to a total on a 0–360 scale. Using the average keeps the math fair across dimensions that have different numbers of criteria (Market has 3, Tech has 1, etc).

      Per dimension
      contribution = avg( criterion_scores ) × weight × 60
      Total VALUEX score
      total = Σ all 7 dimension contributions   → range 0 – 360
      Maximum is exactly 360 (every criterion at 6, every dimension at 6 × weight × 60, weights sum to 1). The historical listing threshold of 200 maps onto this scale — the cut-off you already know still applies.

      9 Contradictions, missing proof & red flags

      Alongside the scores, VALUEX runs three extra passes — the plausibility check you used to do by hand, now automatic and itemised with severity and source.

      Contradictions
      Where two inputs disagree — deck says 25% team allocation, tokenomics doc says 40%. Each gets a severity and a suggested clarification.
      Missing evidence
      Proof that should exist for this stage & sector but isn't there — no legal opinion for a token in regulated jurisdictions, no CAC for a revenue claim.
      Red flags
      Patterns that materially raise listing risk — anonymous founders for a fintech, unaudited contracts holding funds.
